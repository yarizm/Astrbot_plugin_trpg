# AstrBot TRPG 选本插件

这是一个面向 AstrBot 的跑团选本 MVP 插件，支持：

- 管理员两步导入 Markdown 剧本大纲
- 将一份大纲拆分成多个候选剧本
- 审核后发布给玩家查看和选择
- 在群聊中把已发布剧本绑定为当前团的激活剧本
- 内置 3 个官方剧本，并支持首次自动初始化

## 目录结构

- `main.py`：AstrBot 插件入口，负责命令注册、事件适配和权限判断
- `core/parser.py`：Markdown 剧本大纲解析
- `core/store.py`：SQLite 持久化与查询
- `core/service.py`：导入、发布、群会话绑定等业务编排
- `core/builtin_scenarios.py`：插件内置剧本包
- `tests/`：核心逻辑单元测试

## 命令

- `/trpg 导入`
- `/trpg 草稿列表`
- `/trpg 发布 <id>`
- `/trpg 初始化内置剧本`
- `/trpg 剧本列表`
- `/trpg 单人开始 <id>`
- `/trpg 单人状态`
- `/trpg 单人结束`
- `/trpg 选剧本 <id>`
- `/trpg 当前剧本`
- `/trpg 重置当前剧本`

## 内置剧本

插件内置了 3 个可直接使用的剧本：

- `雾港回声`：海港悬疑克苏鲁调查
- `红枫高中的第十三间教室`：校园怪谈推理惊悚
- `星环列车终点站`：太空列车密室悬疑

默认配置下，插件首次启动会自动把这 3 个内置剧本写入数据库并直接发布。

如果关闭了自动初始化，也可以让管理员手动执行：

```text
/trpg 初始化内置剧本
```

## 单人模式

在和机器人的私聊中，可以直接开启单人跑团：

```text
/trpg 剧本列表
/trpg 单人开始 1
```

开始后，直接继续发送自然语言行动即可，例如：

- `我先调查房间里最可疑的痕迹`
- `我去找刚才那个目击者继续追问`
- `我决定直接进入灯塔`

可用命令：

- `/trpg 单人状态`
- `/trpg 单人结束`

## 导入格式

管理员执行 `/trpg 导入` 后，在下一条消息中直接粘贴 Markdown：

```md
## 剧本：雾港回声
### 简介
暴雨夜的白礁港里，一座废弃灯塔重新亮起。
### 标签
悬疑, 克苏鲁, 调查
### 推荐人数
3-5 人
### 开场设定
玩家在港口躲雨时，发现一艘失踪渔船空无一人地漂回码头。

## 剧本：山路终点
### 简介
一辆旅游大巴误入封闭山道，乘客逐渐发现彼此并不陌生。
```

说明：

- 每个剧本必须以 `## 剧本：标题` 开头
- 支持 `### 简介`、`### 标签`、`### 推荐人数`、`### 开场设定`
- 如果没有填写 `简介`，系统会自动回退到首段描述

## 配置

- `admin_user_ids`：允许执行管理命令的用户 ID 列表
- `max_import_chars`：单次导入的最大字符数
- `max_list_size`：列表命令最大返回数量
- `db_filename`：SQLite 数据文件名
- `bootstrap_builtin_scenarios`：是否在插件启动时自动初始化内置剧本
